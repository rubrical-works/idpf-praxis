# Internationalization (i18n) Domain
**Version:** v0.66.3
**Type:** Domain
**Companion Skill:** i18n-setup
## Overview
Evaluative criteria for internationalization and localization across project artifacts. Surfaces string externalization, locale-aware formatting, bidirectional text, and translation workflow requirements during reviews.
**Core Principle:** Internationalization is an architecture concern. String externalization, locale-aware formatting, and RTL support must be designed in from the start.
## Domain Scope
| Concern | Description |
|---------|-------------|
| String Externalization | Resource files, message keys, no hardcoded user-facing strings |
| ICU MessageFormat | Pluralization, gender, number/date formatting with locale awareness |
| Locale-Aware Formatting | Dates, numbers, currencies, time zones per locale |
| Bidirectional Text | RTL language support, CSS logical properties, text alignment |
| Translation Workflow | TMS platforms, key naming conventions, context for translators |
## When This Domain Applies
- Applications targeting multiple languages or locales
- Adding user-facing strings (labels, messages, errors)
- Implementing date, number, or currency formatting
- Supporting RTL languages (Arabic, Hebrew, Farsi)
- Setting up translation workflows
