# SEO Domain
**Version:** v0.64.0
**Type:** Domain
**Companion Skill:** seo-optimization
## Overview
Evaluative criteria for search engine optimization across project artifacts. Surfaces discoverability, crawlability, and structured data requirements during reviews.
**Core Principle:** SEO is an architectural concern. URL structure, rendering strategy, meta tags, and structured data should be decided during design, not added after launch.
## Domain Scope
| Concern | Description |
|---------|-------------|
| Technical SEO | Server-side rendering, URL structure, canonical URLs, redirects, sitemaps, robots.txt |
| Content Optimization | Heading hierarchy, meta tags, semantic HTML, alt attributes, internal linking |
| Structured Data | JSON-LD schema markup, rich results eligibility, Open Graph, Twitter Cards |
| Crawlability | CSR vs SSR tradeoffs, JavaScript rendering, lazy loading |
| Performance | Core Web Vitals (LCP, FID, CLS), page speed, mobile-friendliness |
## When This Domain Applies
- Building web-facing applications needing organic search traffic
- Designing URL structures or routing for public pages
- Choosing rendering strategies (SSR, SSG, CSR, ISR)
- Adding/modifying content pages, blogs, or documentation
- Migrating URLs or restructuring site architecture
