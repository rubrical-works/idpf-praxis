# Contributing to IDPF Praxis

Thank you for your interest in contributing to the IDPF framework!

## How This Repository Works

This is the **distribution repository** — it contains released framework files deployed automatically from a private development repo. Code changes are not accepted directly here.

You can contribute by:
- Reporting bugs
- Requesting features or enhancements
- Requesting to become a contributor
- Improving documentation

## Reporting Bugs

1. Click **New Issue** on this repository
2. Select **Bug Report** from the chooser
3. Fill in the required fields: description, reproduction steps, expected/actual behavior, and environment
4. The template will auto-label your issue as `bug`

**Before filing:** Check [existing issues](https://github.com/rubrical-works/idpf-praxis/issues) to avoid duplicates.

## Requesting Features

1. Click **New Issue** on this repository
2. Select **Enhancement** from the chooser
3. Describe the feature, your motivation, and a proposed solution
4. The template will auto-label your issue as `enhancement`

## Becoming a Contributor

If you'd like to contribute code, documentation, or skills to the framework:

1. Click **New Issue** on this repository
2. Select **Request to be a Contributor** from the chooser
3. Tell us about your background, areas of interest, and availability

We'll review your request and follow up with next steps.

## Requesting Praxis Hub Manager

Praxis Hub Manager is the desktop application for installing and managing IDPF:

1. Click **New Issue** on this repository
2. Select **Request Download for Praxis Hub Manager** from the chooser
3. Select your platform and describe your use case

## Security Vulnerabilities

**Do not report security vulnerabilities through public issues.** See [SECURITY.md](SECURITY.md) for responsible disclosure instructions.

## Release Process

Releases are tagged with semantic versioning (`vX.Y.Z`). Each release includes:
- Full release notes on the [Releases page](https://github.com/rubrical-works/idpf-praxis/releases)
- A changelog in [CHANGELOG.md](CHANGELOG.md)
- Skill catalog and checksum assets

To stay up to date, use the hub update feature in Praxis Hub Manager.

## Questions?

Open a [Discussion](https://github.com/rubrical-works/idpf-praxis/discussions) for general questions, ideas, or feedback.
